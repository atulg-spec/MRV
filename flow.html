Decentralized Storage: IPFS

Media, verification reports, and raw datasets stored off-chain with blockchain-stored CIDs.

  • Content-addressed storage for permanence
  • Reduces on-chain gas costs
  • IPFS Cluster for pinning reliability
  • Production: multiple pinning services

function registerRestoration(
    string memory _cid,
    uint256 _area,
    uint256 _carbonEstimate
) external onlyVerifier returns (uint256) {
    projectCount++;
    
    projects[projectCount] = RestorationProject({
        id: projectCount,
        cid: _cid,
        area: _area,
        carbonEstimate: _carbonEstimate,
        registeredAt: block.timestamp,
        verifiedBy: msg.sender
    });
    
    emit ProjectRegistered(
        projectCount,
        _cid,
        _area,
        _carbonEstimate,
        msg.sender
    );
    
    return projectCount;
}
JavaScript 
// Store data with IPFS and record on blockchain
async function registerProject(projectData) {
    try {
        // Add to IPFS
        const cid = await ipfs.add(JSON.stringify(projectData));
        
        // Store reference on blockchain
        const tx = await contract.registerRestoration(
            cid.path,
            projectData.area,
            projectData.carbonEstimate
        );
        
        // Wait for transaction confirmation
        const receipt = await tx.wait();
        
        return {
            success: true,
            cid: cid.path,
            txHash: receipt.transactionHash
        };
    } catch (error) {
        return {
            success: false,
            error: error.message
        };
    }
}

Prototype vs. Production Note

This demonstration uses simulated data and localStorage for demonstration purposes. In a production environment:

  • IPFS would be used for decentralized storage instead of browser localStorage
  • Polygon mainnet or a secure L2 would be used instead of simulated transactions
  • All data would be immutable and permanently stored
  • Smart contracts would be audited and deployed with proper security measures

Frequently Asked Questions

Data immutability is achieved through a combination of blockchain anchoring and decentralized storage. All critical records are stored on IPFS with their content identifiers (CIDs) written to the Polygon blockchain. Once recorded, these CIDs cannot be altered, ensuring the permanence and verifiability of the stored data.

The prototype uses browser localStorage for demonstration purposes, which can be cleared. In the production system, data would be stored on IPFS with multiple pinning services ensuring persistence. Once added to IPFS and recorded on blockchain, data becomes immutable and permanent through content addressing and cryptographic hashing.

The system utilizes a hybrid approach where large datasets (like drone imagery) are stored on IPFS, while only the essential metadata and content identifiers are stored on-chain. This approach minimizes gas costs while maintaining data integrity. For processing, we use off-chain computation with oracle services to handle resource-intensive analytics.

Carbon credits are calculated using established methodologies (e.g., VCS, CDM) based on field measurements, remote sensing data, and biome-specific models. Verification is performed by accredited third parties whose signed-off reports are stored on IPFS with their hashes recorded on-chain. This creates a transparent and auditable record of the verification process.

Drones & Sensors
